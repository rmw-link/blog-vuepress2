# Quickjs:n muokkaaminen ruosteen funktioiden tuomiseksi - uusi tapa ajatella Krakenin laajentamista.

## Koodivarasto

* [github](https://github.com/rmw-lib/quickjs-rust)
* [gitee](https://gitee.com/rmw-link/quickjs-rust)
* [gitflic](https://gitflic.ru/project/rmw-link/quickjs-rust)
* [bitbucket](https://bitbucket.org/rmw-link/quickjs-rust)

## Syntyprosessi

[Kraken](https://openkraken.com) on suorituskykyinen web-renderöintimoottori, joka perustuu osoitteeseen `Flutter` ja käyttää skriptausmoottorina [quickjs:ää](https://github.com/openkraken/kraken/tree/main/bridge/third_party/quickjs).

Halusin kirjoittaa joitakin laajennuksia Krakeniin käyttäen `rust`.

Kraken [tukee laajennusten kirjoittamista osoitteessa `dart`](https://openkraken.com/guide/advanced/custom-js-api) .

Käyttämällä [`flutter_rust_bridge`](https://github.com/fzyzcjy/flutter_rust_bridge) `rust` ja `dart`.

Kun nämä kaksi kohtaa yhdistetään, ei ole vaikeaa kirjoittaa Kraken-laajennuksia käyttämällä `rust`.  
Tämän ratkaisun suorituskykyyn liittyvät kustannukset tuntuvat kuitenkin suurilta, sillä `dart` kutsuu `rust` ja `quickjs` kutsuu `dart`, mikä aiheuttaa suorituskyvyn heikkenemistä.

Toisaalta, vaikka `rust` -yhteisö on ollut [`rquickjs`](https://github.com/DelSkayn/rquickjs) tällaiset kutsut `quickjs` -kirjastoon osoitteessa `rust`.  
Ne kuitenkin kutsuvat `quickjs` sen sijaan, että ne upottaisivat `quickjs`, eikä niitä voida käyttää `quickjs` taikomiseen.

Tässä koodipohjassa olen toteuttanut uuden ratkaisun: `quickjs` lähdekoodia on muutettu suoraan tukemaan `rust` -laajennusta.

Tämä on yleinen ratkaisu, jota voidaan käyttää Krakenin muokkaamisen lisäksi myös kaikkiin kehyksiin ja kirjastoihin, jotka upottavat `quickjs`.

### Fib:n toteuttaminen ruosteessa

Js:ssä tuotu fib-funktio on peräisin osoitteesta `rust/src/export/fib.rs` ja koodi on seuraava :

```rust
use crate::js::{self, arg};
use quickjs_ffi::{JSContext, JSValue};
use rust_macro::js;
use std::os::raw::c_int;

#[js]
pub fn fib(n: i64) -> i64 {
  if n <= 1 {
    return if n == 1 { 1 } else { 0 };
  }
  n + fib(n - 1)
}

#[no_mangle]
pub extern "C" fn js_fib(
  ctx: *mut JSContext,
  _this: JSValue,
  argc: c_int,
  argv: *mut JSValue,
) -> JSValue {
  if let Err(err) = arg::arg_miss(ctx, argc, fib_args_len) {
    return err;
  }
  match arg::arg_i64(ctx, argv, 0) {
    Err(err) => err,
    Ok(n) => js::val(ctx, fib(n)),
  }
}
```

Tällä hetkellä proseduurimakro `#[js]` lisää vain vakion `fib_args_len`, joka määrittää funktion argumenttien määrän.

Tulevaisuudessa voidaan kirjoittaa menettelymakro `./rust_macro`, joka mahdollistaa täysin automaattisen toimintojen viennin.

      throw(ctx, format!("not number : args position {pos}"))?

    let mut n = MaybeUninit::uninit();

Tällä hetkellä tarkistetaan vain argumenttien lukumäärä ja luetaan i64-tyyppi.

Voit lisätä näitä funktioita tarpeen mukaan, katso [qjs_sys](https://docs.rs/qjs-sys/0.1.2/qjs_sys/):n funktiot alkaen osoitteesta `JS_To`.

## Hakemiston rakenne

* `./quickjs_rust`  
  Quickjs-koodin c-tiedoston muuttaminen
  
* `./quickjs_ffi`  
  Vie funktiot `quickjs` otsikkotiedostosta osoitteeseen `rust`

  Käytä `rust` toteuttamaan `quickjs`:n toiminnot.
  
  * `./rust/src/qjs.rs`

```xonsh

from pathlib import Path

cd @(PWD)

p".xonshrc".exists() && source .xonshrc

quickjs = 'quickjs'

if not exists(quickjs):
  git clone git@github.com:bellard/@(quickjs).git --depth=1

./rust/build.xsh

def ln_s(li):
  for arg in li.split(' '):
    fp = join(quickjs,arg)
    if not exists(fp):

cd @(quickjs)

cd @(PWD)
./quickjs/qjs --unhandled-rejection -m test.js 2>&1 | tee test.js.out
```

## Periaatteen selitys

### `quickjs_rust/patch.py`

Käynnistämällä `./quickjs_rust/patch.py` tehdään joitakin pieniä muutoksia `quickjs` -lähdekoodiin.

Yksi funktioista `JS_AddRust` käytetään ruoste-moduulin ruiskuttamiseen.

`rust_run` lisätään osoitteeseen `JS_ExecutePendingJob` asynkronisten toimintojen kutsumiseksi.

### `quickjs_rust.h`

Yllä olevista muutoksista näet, että olemme ottaneet käyttöön uuden otsikkotiedoston `quickjs_rust.h`, jossa on seuraava koodi

```c
#ifndef QUICKJS_RUST_H
#define QUICKJS_RUST_H

#include "../quickjs/quickjs.h"
#include "../rust/rust.h"

#define countof(x) (sizeof(x) / sizeof((x)[0]))
#define JS_RUSTFUNC_DEF(name) JS_CFUNC_DEF(#name, name##_args_len, js_##name)
#include "./js_rust_funcs.h"

static const unsigned int js_rust_funcs_count = countof(js_rust_funcs);

static int
js_rust_init(JSContext* ctx, JSModuleDef* m)
{
  return JS_SetModuleExportList(ctx, m, js_rust_funcs,
      js_rust_funcs_count);
}

#define JS_INIT_MODULE js_init_module_rust

JSModuleDef* JS_INIT_MODULE(JSContext* ctx, const char* module_name)
{
  JSModuleDef* m;
  m = JS_NewCModule(ctx, module_name, js_rust_init);
  if (!m)
    return NULL;
  js_rust_init(ctx, m);
  return m;
}

void JS_AddRust(JSContext* ctx, JSRuntime* rt)
{
  JSModuleDef* m = JS_INIT_MODULE(ctx, "rust");
  for (unsigned int i = 0; i < js_rust_funcs_count; i++) {
    JS_AddModuleExport(ctx, m, js_rust_funcs[i].name);
  }
  rust_init(ctx, rt);
}

#endif
```

### `rust/rust.h`

Näet, että `quickjs_rust/quickjs_rust.h` esittelee `quickjs_rust/js_rust_funcs.h`, joka luodaan automaattisesti rust-vientifunktion otsikkotiedostosta `rust/rust.h`, eikä sitä pitäisi muuttaa käsin.

Ja `rust/rust.h` luodaan kutsumalla cbindgen osoitteesta `./rust/build.xsh`.

### `rust/build.xsh`

```xonsh
#!/usr/bin/env xonsh

from os.path import dirname,abspath
import platform
PWD = dirname(abspath(__file__))
cd @(PWD)

p"../.xonshrc".exists() && source ../.xonshrc

./src/export/mod.gen.py

system = platform.system().lower()
if system == 'darwin':
  system = f'apple-{system}'

TARGET=f'{platform.machine()}-{system}'

def cbindgen():
  cbindgen -q --config cbindgen.toml --crate rust --output rust.h

try:
  cbindgen()
except:
  cargo clean
  cbindgen()

cargo build \
--release \
-Z build-std=std,panic_abort \
-Z build-std-features=panic_immediate_abort \
--target @(TARGET)

mv ./target/@(TARGET)/release/librust.a ./target/release
```

pub(crate) fn arg_i64(ctx: *mut JSContext, argv: *mut JSValue, pos: isize) -> Result<i64, JSValue> {
  unsafe {
    let val = *argv.offset(pos);
    if JS_IsNumber(val) == 0 {
      throw(ctx, format!("not number : args position {pos}"))?
    }
    let mut n = MaybeUninit::uninit();
    JS_ToInt64(ctx, n.as_mut_ptr() as _, val);
    Ok(n.assume_init())
  }
